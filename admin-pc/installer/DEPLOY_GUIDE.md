# คู่มือติดตั้ง LibraHub (สำหรับผู้ติดตั้ง)

คู่มือนี้เขียนให้ทำตามได้แม้ไม่เชี่ยวชาญคอมพิวเตอร์ ทำทีละขั้นตามลำดับ

---

## 1. ภาพรวมระบบ

**LibraHub** มี 3 ส่วนทำงานร่วมกัน:

| ส่วน | ทำอะไร | ติดตั้งที่เครื่องไหน |
|---|---|---|
| **Server** (สมองของระบบ) | เก็บข้อมูลหนังสือ + แจกจ่ายให้ทุกจอ | **เครื่องบรรณารักษ์** |
| **Admin** (โปรแกรมผู้ดูแล) | เพิ่ม/แก้หนังสือ, ปก, โลโก้, ตั้งค่า | **เครื่องบรรณารักษ์** (เครื่องเดียวกับ Server) |
| **Kiosk** (จอค้นหา) "LibraHub Search" | ให้นักเรียน/ผู้ใช้ค้นหาหนังสือ | **จอสัมผัสแต่ละจอ** |

**สำคัญ:** ทุกเครื่องต้องอยู่ใน **เครือข่าย (Wi‑Fi/LAN) วงเดียวกัน** กับเครื่องบรรณารักษ์ จอ Kiosk จึงจะคุยกับ Server ได้

```
        เครื่องบรรณารักษ์                       จอ Kiosk (ติดตั้งได้หลายจอ)
   ┌───────────────────────┐               ┌──────────┐  ┌──────────┐
   │  Server  +  Admin     │◄── LAN วงเดียว ──►│ Kiosk #1 │  │ Kiosk #2 │ ...
   └───────────────────────┘               └──────────┘  └──────────┘
```

มี **ตัวติดตั้ง 2 ตัว**:
- `LibraHubAdminServer_Setup_x.x.x.exe` → ลงที่เครื่องบรรณารักษ์
- `LibraHubKiosk_Setup_x.x.x.exe` → ลงที่จอ Kiosk ทุกจอ

> ไม่ต้องติดตั้ง .NET หรือโปรแกรมเสริมใด ๆ — ตัวติดตั้งรวมทุกอย่างมาให้แล้ว (self‑contained)

---

## 2. เตรียมก่อนติดตั้ง

- เครื่องทุกเครื่องเป็น **Windows 64‑bit (x64)**
- ทุกเครื่องต่อ **เครือข่ายวงเดียวกัน**
- มีไฟล์ติดตั้ง 2 ตัวข้างต้น

### (เฉพาะผู้ที่ต้อง "สร้าง" ตัวติดตั้งเอง — ข้ามได้ถ้ามีไฟล์ .exe แล้ว)
ทำบนเครื่องของผู้พัฒนา/ผู้ build:
1. ติดตั้ง **Inno Setup 6** (โปรแกรมสร้างตัวติดตั้ง) — https://jrsoftware.org/isdl.php
2. เปิด PowerShell ที่โฟลเดอร์ `admin-pc\installer` แล้วรัน:
   ```
   powershell -ExecutionPolicy Bypass -File build-release.ps1
   ```
   จะได้โฟลเดอร์ `dist\Server`, `dist\AdminPC`, `dist\Kiosk` + ไฟล์ `LibraHub.ico`
3. สร้างตัวติดตั้งทั้งสอง:
   ```
   iscc LibraHubAdminServer.iss
   iscc LibraHubKiosk.iss
   ```
   ไฟล์ `.exe` จะอยู่ในโฟลเดอร์ `Output\`
   - หมายเหตุ: ต้องมีไฟล์ `license_keys_10000.csv` (รายการรหัสลิขสิทธิ์) วางตามที่ `build-release.ps1` กำหนด มิฉะนั้นจะขึ้นเตือน
   - ถ้าใช้ **Inno Setup 6.3 ขึ้นไป** ให้แก้ในไฟล์ `.iss` ทั้งสอง จาก `ArchitecturesAllowed=x64` เป็น `ArchitecturesAllowed=x64compatible`

---

## 3. ติดตั้งเครื่องบรรณารักษ์ (Admin + Server)

1. คลิกขวาที่ `LibraHubAdminServer_Setup_x.x.x.exe` → **Run as administrator** (เปิดแบบผู้ดูแล)
   - จำเป็น เพราะตัวติดตั้งต้องตั้งค่า Firewall ให้จอ Kiosk เข้ามาได้
2. กด Next ไปตามขั้นตอนจนติดตั้งเสร็จ
3. **หน้าจบการติดตั้งจะแสดง "AdminKey"** (รหัสลับของระบบ) เช่น `H8oIzURx...=`
   - 📋 **คัดลอกเก็บไว้** — รหัสนี้ถูกบันทึกไว้ในไฟล์ **`AdminKey.txt`** ในโฟลเดอร์ที่ติดตั้งด้วย (ปกติ `C:\Program Files\LibraHub\AdminKey.txt`)
4. กรอก AdminKey เข้าโปรแกรม Admin:
   - เปิด **LibraHub Admin** (ไอคอนบน Desktop / เมนู Start)
   - ไปที่ **ตั้งค่า (Settings)** → ช่อง **AdminKey** → **วางรหัสที่คัดลอกมา** → กด **บันทึก**
   - ⚠️ รหัสต้อง **ตรงกัน** ระหว่าง Admin กับ Server มิฉะนั้นการเพิ่ม/แก้/ลบหนังสือจะไม่ทำงาน
5. 🔒 **ความปลอดภัย:** AdminKey คือกุญแจของระบบ — เก็บไฟล์ `AdminKey.txt` ไว้ในที่ปลอดภัย หรือ **ลบทิ้งหลังกรอกเสร็จ** (ถ้าลืมรหัส ติดตั้งซ้ำจะ *ไม่* สร้างใหม่ ให้เปิดดูที่ `AdminKey.txt` หรือเปิดไฟล์ `Server\appsettings.json`)
6. **เปิดใช้งานลิขสิทธิ์ (License):**
   - เปิด LibraHub Admin → เมนูลิขสิทธิ์ → ใส่ Product Key (ผูกกับเมนบอร์ดเครื่องนี้)
   - ถ้าระบบต้องใช้ไฟล์รายการรหัส (`license_keys_10000.csv`) ให้แน่ใจว่ามีอยู่ในโฟลเดอร์ `Server` (ปกติมากับตัวติดตั้งแล้ว)

> เริ่มใช้งาน: เปิดด้วยทางลัด **"เปิดระบบ LibraHub (Server + Admin)"** จะเปิด Server แล้วตามด้วย Admin ให้อัตโนมัติ

## 4. ติดตั้งจอ Kiosk (ทำทุกจอ)

1. รัน `LibraHubKiosk_Setup_x.x.x.exe` (ไม่ต้อง Run as administrator — ติดตั้งเฉพาะผู้ใช้)
2. ติดตั้งเสร็จจะมีไอคอน **"LibraHub Search"**
3. เปิด **LibraHub Search** → แตะที่ปุ่ม **ตั้งค่า (รูปเฟือง)** → ใส่ **PIN เริ่มต้น `1234`**
4. ตั้งค่าให้ครบ:
   - **Server URL:** ปกติจอจะหา Server เจอเอง (อัตโนมัติผ่านระบบ mDNS)
     - ถ้าหาไม่เจอ ให้พิมพ์เอง: `http://<IP ของเครื่องบรรณารักษ์>:45269`
     - **วิธีหา IP เครื่องบรรณารักษ์:** ที่เครื่องบรรณารักษ์ เปิด Command Prompt พิมพ์ `ipconfig` กด Enter → ดูเลข **IPv4 Address** เช่น `192.168.1.50` → URL จะเป็น `http://192.168.1.50:45269`
   - **การวางจอ (orientation):** เลือก **แนวตั้ง** หรือ **แนวนอน** ตามการติดตั้งจอจริง
   - **เปลี่ยน PIN:** ตั้ง PIN ใหม่แทน `1234` (กันคนอื่นเข้าตั้งค่า)
   - **ธีมสี / ชื่อระบบ / พื้นหลัง:** ปรับได้ตามต้องการ
5. **เปิดอัตโนมัติเมื่อเปิดเครื่อง:** ตัวติดตั้งตั้งให้แล้ว (จอจะเปิด LibraHub Search เองหลังเข้าระบบ Windows) — ตรวจ/ปิด‑เปิดได้ในหน้า **ตั้งค่า**
6. จอจะแสดง **เต็มจอ** อัตโนมัติ (โหมด Kiosk)

> ทำซ้ำขั้นตอนนี้กับจอ Kiosk ทุกจอ ทุกจอชี้ไปที่ Server เครื่องเดียวกัน

---

## 5. การใช้งานหลายจอ

- ติดตั้ง **LibraHub Search** ได้หลายจอ ทุกจอชี้ Server เดียวกัน
- ดูจำนวนจอที่กำลังเชื่อมต่อได้ที่ **LibraHub Admin → Dashboard → "จอ Kiosk เชื่อมต่อ"** (แสดงเป็น X/10)

---

## 6. ปัญหาที่พบบ่อย + วิธีแก้

**จอ Kiosk ขึ้น "เชื่อมต่อเซิร์ฟเวอร์ไม่ได้"**
1. เครื่องบรรณารักษ์เปิด **Server** อยู่หรือไม่ (เปิดด้วยทางลัด "เปิดระบบ LibraHub")
2. **URL/IP ถูกไหม** — ลองพิมพ์ `http://<IP server>:45269` ใหม่ (ดูวิธีหา IP ข้อ 4)
3. **Firewall / พอร์ต 45269** — ตัวติดตั้งเปิดให้แล้ว; ถ้าเคยปิด/แก้เอง ให้เปิดใหม่ (ดูข้อ 8)
4. ทุกเครื่องอยู่ **เครือข่ายวงเดียวกัน** หรือไม่ (Wi‑Fi/สาย LAN เดียวกัน)

**ลืม PIN ของจอ Kiosk**
- ที่เครื่องบรรณารักษ์: **LibraHub Admin → ตั้งค่า → รีเซ็ต PIN จอ Kiosk** → PIN ทุกจอที่เชื่อมต่ออยู่จะกลับเป็น `1234` → ไปตั้งใหม่ที่จอ
- (ต้องตั้ง AdminKey ให้ตรงกันก่อน ดูข้อ 3)

**กด Admin แล้วขึ้นพลาด / เพิ่ม‑ลบหนังสือไม่ได้ (Error 401)**
- AdminKey **ไม่ตรงกัน** — ตรวจให้ AdminKey ในโปรแกรม Admin ตรงกับของ Server (ดูที่ `AdminKey.txt` หรือ `Server\appsettings.json`)

**ต้องการเปลี่ยนพอร์ต (ถ้า 45269 ชนกับโปรแกรมอื่น)**
- แก้ 3 จุดให้ตรงกัน: (1) `Server\appsettings.json` ช่อง `Urls` → `http://0.0.0.0:<พอร์ตใหม่>` (2) กฎ Firewall (ดูข้อ 8) (3) Server URL ในจอ Kiosk ทุกจอ

---

## 7. ถอนโปรแกรมเก่า (ถ้าเคยมี "LibrarySystem")

LibraHub เป็นคนละโปรแกรมกับ "LibrarySystem" ตัวเดิม — ก่อนติดตั้ง LibraHub ให้:
1. ไปที่ **Settings → Apps → Installed apps** (หรือ Control Panel → Programs and Features)
2. หา **LibrarySystem** → **Uninstall** (ถอนออก)
3. แล้วค่อยติดตั้ง LibraHub ตามคู่มือนี้

(ข้อมูลหนังสือเดิมอยู่ที่ `C:\ProgramData\LibrarySystem\library.db` — ถ้าต้องการเก็บข้อมูลเดิม ห้ามลบโฟลเดอร์นี้)

## 8. หมายเหตุเทคนิค (สำหรับผู้ดูแลระบบ)

- **พอร์ต:** Server ฟังที่ `0.0.0.0:45269` (ค่าเริ่มต้นใหม่ เลี่ยงการชนกับระบบอื่น)
- **กฎ Firewall (เปิด/ปิดเอง):** เปิด Command Prompt แบบ Administrator
  - เปิด: `netsh advfirewall firewall add rule name="LibraHub Server 45269" dir=in action=allow protocol=TCP localport=45269`
  - ลบ: `netsh advfirewall firewall delete rule name="LibraHub Server 45269"`
- **AdminKey อ่านจาก** `Server\appsettings.json` ตอน Server เริ่มทำงาน — Server **ต้องรันโดยมี working directory = โฟลเดอร์ Server** (ทางลัด/`StartAll.bat` ตั้งให้ถูกแล้ว) ถ้ารันจากที่อื่น AdminKey จะไม่ถูกอ่าน
  - ทางเลือก: ตั้งเป็น Environment Variable ชื่อ `AdminKey` (ไม่ขึ้นกับ working directory)
- **mDNS:** Server ประกาศตัวเองในวง LAN ชื่อบริการ `_libraryapi._tcp` พอร์ตตรงกับที่ฟังจริง จอ Kiosk จึงค้นหาเจออัตโนมัติ
- **Self‑contained:** ทุกแอปรวม .NET runtime มาในตัว — เครื่องลูกค้า **ไม่ต้องติดตั้ง .NET** แยก
- **ที่เก็บข้อมูล:** ฐานข้อมูล + การตั้งค่า Server อยู่ที่ `C:\ProgramData\LibrarySystem\` ; การตั้งค่าจอ Kiosk อยู่ที่ `C:\ProgramData\LibrarySystem\kiosk-settings.json` (ของแต่ละจอ — ติดตั้งซ้ำไม่ทับค่าที่ตั้งไว้)
- **ค่าเริ่มต้นจอ Kiosk:** เปิดครั้งแรกจะสร้างไฟล์ตั้งค่าเอง (เต็มจอ + พอร์ต 45269) — ติดตั้งใหม่ไม่ทับ KioskId/URL/orientation/ธีม ที่เคยตั้ง
